浅谈响应式网页设计

发表于 2015-05-10

从接触前端以来，一直有听到所谓的响应式网页设计，但实际上对其概念也只是一知半解，大概只知道响应式网页设计就是能在各种分辨率下网页的依然能有良好的表现效果。刚好今年在做工作室的招新网站的时候，刚好面临响应式网页设计的需求，借此机会对响应式网页设计有了深一点的理解。如有写得不好或者不对的地方，欢迎留言补充，谢谢指教！

有几种响应式的方法？

###1. 使用 CSS3 的媒体查询
这种方法是使用 CSS3 进行媒体查询，对应不同的分辨率设置不同的 CSS 属性。举个栗子：

@media screen and (max-width: 960px){
    body {
        background: #000;
    }
}

这段代码的意思就是在页面小于 960 px 的时候执行 {} 内的 CSS 代码。

另外值得一提的是，上述代码中添加的 screen 是指当设备在打印页面时使用衬线字体，在屏幕上显示时用无衬线字体。因此，如果所写的页面不考虑打印的情况的话，可以去掉它。

由此我们可以知道当使用这种方法时，我们需要考虑许多种不同分辨率的情况。

所以使用这种方法的优点有：
1.针对性。对不同分辨率的设备有不同的预定义的一套 CSS 代码
2.易用性。这种方法通俗易懂，出现了哪些分辨率的问题可以很快想到解决办法
3.主流。这是目前应用最多的自适应网页设计方案

缺点有：
1.复杂性。需要对不同设备率分辨率编写多套 CSS 代码。越细致的自适应网页可能代码数量越庞大，这会导致不易维护
2.需要多套设计图。对设计师的要求较高，设计师需要对不同分辨率的设备出几套设计图。

###2.固定宽高比
招新网页用的就是这种方法。其大概意思就是，将页面主要内容（背景可占满整个页面）限定在一个固定的宽高比例中，页面内容的元素的定位，大小等全部用百分比的形式来描述，这样就实现了任意分辨率下，页面内容都是等比例缩小或放大的。

限定页面内容的宽高比例用到了 Javascript 去动态的检测设备的分辨率。

现将找新页面检测分辨率的代码抽象，如下:

首先设置页面内容的类名为 animate-container
<div class="animate-container">

Javascript 代码

function fixedPage(x,y) {
  var oHeight = $window.height(),//窗口高度
oWidth = $window.width(),//窗口宽度
      cWidth = oHeight * (x/y),//等比例宽度
      oMargin = (oWidth - cWidth)/2,//左右外间距
      $container = $('.animate-container');
  $container.css({'width':cWidth,'height':oHeight,'left':oMargin});//设置高度为窗口高度，宽度为等比例缩放宽度，并居中显示
}

最后，招新项目中图片的宽高比为 750/1206
所以
fixedPage(750,1206);
